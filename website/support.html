Run a quick health check

vmux doctorchecks the local environment and session state
vmux statusshows whether the vmux daemon is reachable
vmux relay statuschecks the phone relay
tailscale statusconfirms both devices are on the tailnet

Remote cannot connect

  1. Confirm Tailscale is connected on both the host and phone.
  2. Run vmux relay serve on the host and keep it running.
  3. Use the address returned by tailscale ip -4 and port 4399.
  4. Check reachability with curl -s http://$(tailscale ip -4):4399/v1/health.

Include these details

Tell us your vmux version, operating system, install method, the command you ran, and any relevant terminal output. Please remove secrets, device tokens, private hostnames, and terminal contents before sharing logs.

Lost a paired phone?

List paired devices with vmux relay devices list, then revoke it with vmux relay devices revoke <device_id>.
